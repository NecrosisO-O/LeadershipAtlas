# 当前测试体系说明

## 1. 测试目标

当前测试的目标不是证明系统已经能在所有场景下精确命中某一个具体领导人，而是回答以下问题：

1. 在 `完整答卷` 场景下，系统是否稳定
2. 在 `相邻人物` 场景下，系统是否能够拉开人物差异
3. 在 `稀疏鲜明特征` 场景下，系统的上限和问题在哪里

## 2. 当前测试结构

### 第一阶段：自我回归测试

- 输入：17位领导人的完整 `profile_data.json`
- 目的：检查每位领导人是否能把自己匹配回前列
- 当前定位：`基础正确性测试`

### 第二阶段：人工构造答卷测试

- 输入：围绕若干人物类型构造的测试答卷
- 目的：观察系统在“鲜明特征组合”场景下会把用户吸向谁
- 当前定位：`问题暴露测试`

### 第二阶段补充测试

- 输入：更针对性的鲜明特征构造答卷
- 目的：检查高相似人物簇是否仍难区分
- 当前定位：`针对性补充测试`

### 第三阶段：相邻人物区分测试

- 输入：已知高相似人物对照组
- 目的：检查完整人物答卷之间是否能分开
- 当前定位：`近邻区分测试`

## 3. 当前对测试结果的解释原则

### 第一阶段

- 是主参考测试
- 若该阶段稳定，说明人物库与基础匹配逻辑成立

### 第二阶段 / 第二阶段补充测试

- 当前不再视为“产品真实命中率”的直接代表
- 更适合视为：
  - `压力测试`
  - `人物簇吸附问题暴露测试`

### 第三阶段

- 是判断相邻人物是否可区分的重要参考
- 若第三阶段整体可区分，说明系统在完整输入场景下仍有可用性

## 4. 当前测试体系结论

一句话总结：

- `完整答卷场景是当前测试体系的主判断依据`
- `稀疏鲜明特征场景主要用于暴露问题，不再作为是否可用的唯一判断标准`

- 早期测试方案曾把人工构造答卷测试当作主测试使用。
- 当前口径已调整：
  - 第一阶段 + 第三阶段更能代表当前系统在完整输入场景下的表现

- 如需复核当前结果，应以 `run_matching_tests.py` 再生成结果并与上述结果文件对照。
