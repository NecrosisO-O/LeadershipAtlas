# 匹配规则（可执行版）

## 一、匹配对象

- 用户答卷 vs 领导人 `profile_data.json`
- 匹配基本单位是 `题目`
- 计算时先算 `题目相似度`，再汇总成 `维度相似度`

## 二、题目相似度

### 1. Likert 题（1-7）

公式：

```text
题目相似度 = 1 - |用户答案 - 领导人答案| / 6
```

范围：0-1

### 2. 单选题 / 强制权衡题

公式：

```text
相同 = 1
不同 = 0
```

### 3. 点数分配题

设总差异为各选项绝对值差之和。

```text
题目相似度 = 1 - 总差异 / 20
```

说明：当前点数总额为 10，四项分配下最大总差异为 20。

## 三、维度相似度

同一维度下，取所有已纳入题目的题目相似度平均值：

```text
维度相似度 = 该维度全部题目相似度平均值
```

## 四、核心维度与参考维度得分

对每位领导人分别计算：

- 核心维度平均相似度
- 参考维度平均相似度

基础得分公式：

```text
基础得分 = 0.8 × 核心维度平均相似度 + 0.2 × 参考维度平均相似度
```

## 五、反向扣分

只针对 `核心维度`。

- 若某核心维度相似度 `< 0.35`，记为 `严重不匹配`
- 若某核心维度相似度 `>= 0.35 且 < 0.55`，记为 `中度不匹配`

计算：

```text
严重不匹配率 = 严重不匹配核心维度数 / 核心维度总数
中度不匹配率 = 中度不匹配核心维度数 / 核心维度总数
扣分 = 0.25 × 严重不匹配率 + 0.10 × 中度不匹配率
```

最终层级得分：

```text
层级最终得分 = 基础得分 - 扣分
若小于 0，则记为 0
```

## 六、风格分、理念分、综合分

### 1. 风格分

- 仅使用 `item_dimension_mapping.json` 中 `layer = style` 的题目

### 2. 理念分

- 仅使用 `item_dimension_mapping.json` 中 `layer = ideology` 的题目

### 3. 综合分

```text
综合分 = 0.5 × 风格分 + 0.5 × 理念分
```

## 七、排序规则

### 默认排序

- 按 `综合分` 从高到低排序

### 同分时顺序

1. 核心维度平均相似度更高者优先
2. 严重不匹配核心维度数更少者优先
3. 风格分更高者优先
4. 理念分更高者优先

## 八、输出层级

至少输出：

- 风格最像谁（Top 3）
- 理念最像谁（Top 3）
- 综合最像谁（Top 3）

## 九、结果解释原则

对综合第一名，提取：

- 最接近的 3 个核心维度
- 最不接近的 1-2 个核心维度

用于结果页解释“为什么像 / 哪里不像”。
