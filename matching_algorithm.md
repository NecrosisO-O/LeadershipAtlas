# 匹配规则（执行同步版）

说明：

- 本文件用于同步说明当前执行真源。
- 当前执行真源是 `matching_engine.py`。
- `run_matching_tests.py` 当前只保留为兼容层入口。
- 若本文件与脚本、当前 `leader_profiles/*/profile_data.json`、`item_dimension_mapping.json`、`signature_dimensions.json` 冲突，以后者为准。
- 旧测试文件已从当前维护基线中移除，本文件不再引用历史测试结论。
- 当前规则是一版面向趣味项目的温和匹配规则：保留辨识度，但弱化硬门槛和过强惩罚。

## 一、匹配对象

- 用户答卷 vs 领导人 `profile_data.json`
- 匹配基本单位是 `题目`
- 计算时先算 `题目相似度`，再汇总成 `维度相似度`
- 用户只会与该领导人 `answers` 中已纳入的题比较；未纳入题不参与该领导人的得分

## 二、题目相似度

### 1. 数值题（Likert 等 1-7 量表）

```text
题目相似度 = 1 - |用户答案 - 领导人答案| / 6
```

- 下限为 `0`
- 上限为 `1`

### 2. 单选题 / 强制权衡题

```text
相同 = 1
不同 = 0
```

### 3. 点数分配题

设总差异为各选项绝对值差之和。

```text
题目相似度 = 1 - 总差异 / 20
```

- 当前点数总额为 `10`
- 四项分配下最大总差异为 `20`

### 4. 类型不一致处理

- 若一方是点数分配字典、另一方不是字典，则该题相似度记为 `0`

## 三、维度层聚合

- 先按 `item_dimension_mapping.json` 中的 `dimension_id` 聚合题目
- 仅聚合同一层中的题：`style` 或 `ideology`
- 同一维度下，取全部命中题目的题目相似度平均值

```text
维度相似度 = 该维度全部命中题目相似度平均值
```

## 四、板块级题处理

当前有两个板块级维度：

- `G_BLOCK_VALUES`
- `G_BLOCK_LEGIT`

这两个维度在维度平均值算出后，会额外乘以：

```text
block_multiplier = 0.6
```

## 五、维度权重

### 1. 核心维度稀有度权重

- 先统计每个维度在多少位领导人的 `profile_report.md` 中被列为 `核心维度`
- 当前不再使用 `1 / sqrt(core_count)` 的连续放大，而是使用轻量三档修正：

```text
core_count <= 2  -> 1.15
core_count <= 4  -> 1.08
其他            -> 1.00
```

- 越少见的核心维度仍会略微加权，但不会被过度放大

### 2. 签名维度加权

- `signature_dimensions.json` 为每位领导人额外定义若干 `signature dimension`
- 若某维度同时满足：
  - 它是该领导人的核心维度
  - 它也是该领导人的签名维度

则该维度权重再乘：

```text
1.5
```

### 3. 核心 / 参考加权平均

- 核心维度：按 `dim_weight` 聚合；签名核心维度再乘 `1.5`
- 参考维度：只按 `dim_weight` 聚合

## 六、层级基础得分

对 `style` 与 `ideology` 分别计算：

```text
基础得分 = 0.80 × 核心维度加权平均 + 0.20 × 参考维度加权平均
```

若某层没有命中核心或参考维度，则对应平均值记为 `0`

## 七、反向扣分

只针对 `核心维度`：

- `严重不匹配`：核心维度相似度 `< 0.35`
- `中度不匹配`：核心维度相似度 `>= 0.35 且 < 0.55`

```text
严重不匹配率 = 严重不匹配核心维度数 / 核心维度总数
中度不匹配率 = 中度不匹配核心维度数 / 核心维度总数
扣分 = 0.20 × 严重不匹配率 + 0.08 × 中度不匹配率
```

## 八、签名维度软惩罚

签名维度不只参与加权，还会在未命中时触发额外软惩罚。

- `signature hit` 的标准：签名核心维度相似度 `>= 0.55`

设：

- `sig_total` = 该层签名核心维度数
- `sig_hits` = 达到命中标准的签名核心维度数

则：

```text
sig_penalty = 0.08 × ((sig_total - sig_hits) / sig_total)
```

- 若该层没有签名核心维度，则 `sig_penalty = 0`
- 未命中的签名维度越多，额外扣分越多
- 当前不再使用硬性 `0.82` 截顶门槛

## 九、风格分、理念分、综合分

### 1. 风格分

- 仅使用 `item_dimension_mapping.json` 中 `layer = style` 的题目

### 2. 理念分

- 仅使用 `item_dimension_mapping.json` 中 `layer = ideology` 的题目

### 3. 层级最终得分

```text
层级最终得分 = 基础得分 - 核心维度扣分 - 签名维度软惩罚
若结果小于 0，则记为 0
```

### 4. 综合分

```text
综合分 = 0.5 × 风格分 + 0.5 × 理念分
```

## 十、排序规则

当前执行实现按以下顺序排序：

1. `综合分` 从高到低
2. `风格分` 从高到低
3. `理念分` 从高到低

当前脚本没有再额外使用文档版的同分细则。

## 十一、输出层级

至少输出：

- 风格最像谁（Top 3）
- 理念最像谁（Top 3）
- 综合最像谁（Top 3）

## 十二、结果解释现状

- 当前执行真源只负责计算排序结果，不再生成已移除的旧测试产物
- “最接近的 3 个核心维度 / 最不接近的 1-2 个核心维度” 这一解释层目前仍属于后续产品实现事项
- 网站结果解释应以当前执行真源或其经回归验证的等价移植为基础，不应只按摘要文字另写一套评分逻辑
