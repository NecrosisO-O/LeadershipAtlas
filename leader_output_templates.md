# 真实领导人模拟答卷输出模板

本文档规定后续子agent在完成一位真实领导人的模拟答卷时，必须同时输出的两种结果模板：

- `说明型输出`：供人工审阅
- `数据型输出`：供程序使用

---

## 一、说明型输出模板

说明型输出用于：

- 人工审阅
- 讨论和复核
- 记录为什么这样判断

固定模板如下：

```md
# 领导人模拟答卷

## 基本信息
- 姓名：
- leader_id：
- 国家/政体：
- 备注：

## 维度层判断
- 核心维度：
- 参考维度：
- 不纳入维度：

## 核心题答卷
- display_no：
  - item_id：
  - legacy_id：
  - 题干摘要：
  - 模拟答案：
  - 依据时期：
  - 证据类型：
  - 说明：

## 参考题答卷
- display_no：
  - item_id：
  - legacy_id：
  - 题干摘要：
  - 模拟答案：
  - 依据时期：
  - 证据类型：
  - 说明：

## 不纳入说明（可选）
- display_no：
  - item_id：
  - legacy_id：
  - 维度：
  - 原因：
```

说明：

- `display_no`：前台显示号（Q号或连续题号）。
- `item_id`：后台稳定题号。
- `legacy_id`：原题号。
- `leader_id`：领导人统一标识，应与目录名及数据型输出中的 `leader_id` 一致。
- `题干摘要` 只需简短概括，不要求粘贴全文。
- `模拟答案` 必须与正式题单的作答格式兼容。
- `依据时期` 按题目级别填写。
- `证据类型` 可自由表述，不强制固定枚举。

---

## 二、数据型输出模板

数据型输出用于：

- 程序读取
- 后续匹配计算
- 统一存档

固定模板如下：

```json
{
  "leader_id": "",
  "leader_name": "",
  "answers": {
    "L01": {
      "legacy_id": "4-1",
      "tier": "core",
      "value": 0,
      "period": "",
      "evidence_type": "",
      "note": ""
    }
  }
}
```

字段说明：

- `leader_id`：领导人唯一标识（文件夹名/程序主键）
- `leader_name`：领导人姓名
- `answers`：所有被纳入题的记录对象，key 统一使用 `item_id`

每道被纳入题至少包含：

- `legacy_id`：原题号
- `tier`：`core` / `reference`
- `value`：模拟答案
- `period`：该题依据的典型时期
- `evidence_type`：证据类型
- `note`：简短说明

说明：

- `answers` 中不出现的题，默认视为该领导人的 `不纳入题`
- 数据型输出不再单独列出：
  - 核心维度
  - 参考维度
  - 不纳入维度
  - 不纳入题
- 这些信息保留在说明型输出中即可。

---

## 三、模拟答案记录要求

无论说明型输出还是数据型输出，`模拟答案` 都必须与用户真实答卷格式兼容：

- 单端 Likert 题：直接记录分值
- 双端 Likert 题：直接记录分值
- 情境单选题：直接记录 `A/B/C/D`
- 强制权衡题：直接记录选择项
- 点数分配题：直接记录分配结果

不得另造抽象坐标体系。

必须遵守以下硬规则：

- 数据型输出中的 `value` 只能写正式数值、正式选项或正式分配结果。
- 不得写自然语言档位，如：
  - `非常同意`
  - `偏左`
  - `strongly agree`
  - `中间偏右`
- 合法示例：
  - `6`
  - `2`
  - `"A"`
  - `{ "A": 4, "B": 3, "C": 2, "D": 1 }`

---

## 四、使用原则

- 说明型输出与数据型输出必须来自同一份判断结果，不能彼此不一致。
- 说明型输出用于人工审阅，数据型输出用于程序处理。
- 两种输出都必须以 `final_questions.md` 为唯一正式题单依据。
