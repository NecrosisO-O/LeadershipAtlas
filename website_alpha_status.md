# 网站 Alpha 状态说明

本文件用于记录网站 Alpha 阶段当前已经完成的内容、仍待补齐的资产，以及后续 Beta 阶段建议推进的方向。

## 一、Alpha 已完成内容

- 已建立 `website/` 前端工程，技术栈为 `React + Vite + TypeScript`
- 已建立网站专用结构化题库与数据生成脚本
- 已将 `82` 题转换为前端可消费的数据文件
- 已接入 `leader_profiles`、`item_dimension_mapping.json`、`signature_dimensions.json` 的网站侧生成资产
- 已完成网站侧 `matching` 模块，能够输出：
  - `综合最像`
  - `风格最像`
  - `理念最像`
- 已完成四段式主流程：
  - 欢迎页
  - 引导页
  - 答题页
  - 结果页
- 已实现五类题型：
  - 单端 Likert
  - 双端 Likert
  - 情境单选
  - 强制权衡
  - 点数分配
- 已实现关键交互规则：
  - 首次作答自动下一题
  - 返回修改不自动下一题
  - 风格/理念之间轻量过渡页
  - 最后一题手动进入结果
  - 未答完不能进入结果页
- 已完成一版 `社论档案感` 基础视觉系统
- 已通过 `vite build` 构建校验
- 已完成网站侧最小烟测：
  - 题库数量为 `82`
  - 五类题型均成功生成
  - 抽测人物画像自喂时，可回归到自身为主结果

## 二、Alpha 当前仍待补齐内容

- 正式人物照片资产
- 图片来源与许可记录
- 真实匹配理由生成
- 更细结果展开层
- 新一轮正式验证体系
- 更系统的移动端人工体验测试

## 三、已知限制

- 当前结果页使用统一占位肖像，不代表最终上线视觉素材
- 当前结果理由是占位文案，不代表最终解释层
- 当前网站侧匹配引擎已对齐现有真源逻辑，但在新的正式验证体系重建前，不应把现有网站结果表述为“已完成充分验证”

## 四、Beta 建议方向

- 接入正式照片与来源记录
- 为结果页补真实理由生成
- 增加更稳定的匹配回归与网站侧验证脚本
- 梳理 `run_matching_tests.py` 的历史命名问题，并抽离更明确的共享引擎真源
- 优化移动端点数分配题与长文本题的细节体验
- 增加更完整的结果页信息层，如人物摘要、更多相近结果与细分解释
